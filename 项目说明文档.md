# 智能安防报警系统 — PPT 素材文档

---

## 一、项目概述

**项目名称**：基于 STM32 的智能安防报警系统

**核心功能**：通过多种传感器实时监测环境，当检测到异常（障碍物入侵、火焰、温度过高）时，自动触发声光报警。

**主控芯片**：STM32F103C8T6（ARM Cortex-M3 内核，72MHz，64KB Flash）

**开发环境**：Keil MDK-ARM V5 + ST-Link 烧录器

**编程语言**：C 语言（基于 STM32 标准外设库 SPL）

### 5.3 报警逻辑（核心算法）

```c
// 三重检测，任一触发即报警
alarm = 0;

if (obstacle == 0)                        // 条件1：红外避障检测到入侵
    alarm = 1;

if (flame == 0)                           // 条件2：火焰传感器检测到明火
    alarm = 1;

if (dht_ok && temp >= TEMP_ALARM_THRESHOLD) // 条件3：温度超过35℃
    alarm = 1;
```

### 5.4 关键技术点

| 技术点 | 说明 |
|-------|------|
| JTAG 引脚复用 | PA15、PB4 默认为 JTAG 引脚，需通过 AFIO 禁用 JTAG 后才能作为普通 GPIO 使用 |
| DHT11 单总线协议 | 通过精确的微秒级时序控制，在同一根数据线上实现双向通信 |
| 软件 I2C | 使用 GPIO 模拟 I2C 时序驱动 OLED 屏幕，灵活选择引脚 |
| GPIO 模式切换 | DHT11 通信中需要在推挽输出和浮空输入之间快速切换，通过直接操作 CRL 寄存器实现 |
| 蓝灯温度显示 | 正常状态每10秒用蓝灯闪烁次数表示温度（先闪十位，再闪个位） |
| 多传感器融合 | 三种传感器数据综合判断，实现多维度安防监测 |

## 六、系统状态说明

### 6.1 正常状态

| 指示 | 表现 |
|------|------|
| 绿色LED | 常亮 |
| 红色LED | 熄灭 |
| 蜂鸣器 | 不响 |
| 板载蓝灯 | 常亮（每10秒闪烁显示温度） |

### 6.2 报警状态

| 指示 | 表现 |
|------|------|
| 红色LED | 200ms 间隔快速闪烁 |
| 绿色LED | 熄灭 |
| 蜂鸣器 | 持续鸣响 |
| 板载蓝灯 | 与红灯同步交替闪烁 |
| OLED屏幕 | 显示 "!! ALARM !!" |

### 6.3 启动自检流程

| 阶段 | 指示 | 含义 |
|------|------|------|
| 蓝灯快闪 3 次 | PC13 闪烁 | 程序已成功启动 |
| 蓝灯快闪 5 次 | PC13 闪烁 | OLED 初始化完成 |
| 绿灯闪 3 次 | PA1 闪烁 | DHT11 温湿度传感器就绪 |
| 红灯闪 3 次 | PA0 闪烁 | DHT11 初始化失败 |

## 九、项目特色与创新点

1. **多传感器融合检测**：集成红外避障、火焰检测、温湿度监测三种传感方式，实现多维度安防监控，降低漏报率。

2. **声光双重报警**：红色 LED 快速闪烁提供视觉警示，有源蜂鸣器持续鸣响提供听觉警示，双重报警确保不被忽视。

3. **启动自检机制**：系统上电后自动检测各模块工作状态，通过不同颜色 LED 闪烁反馈初始化结果，便于快速定位故障。

4. **蓝灯温度编码显示**：创新性地利用板载 LED 闪烁次数编码显示温度值（十位+个位），在没有显示屏的情况下也能获取温度信息。

5. **实时 OLED 信息显示**：通过 OLED 屏幕实时显示温度、湿度、系统状态和各传感器触发情况，信息直观清晰。

6. **低功耗设计**：所有模块采用 3.3V 供电，兼容 USB 供电方式，适合长时间运行。

7. **模块化软件架构**：传感器驱动、显示驱动、延时函数分别封装为独立模块，代码结构清晰，易于维护和扩展。

## 十一、项目总结

本项目基于 STM32F103C8T6 微控制器，设计并实现了一套智能安防报警系统。系统集成了红外避障传感器、火焰传感器和 DHT11 温湿度传感器，能够实时监测环境中的入侵、火灾和温度异常等安全隐患。当任一传感器触发报警条件时，系统通过红色 LED 闪烁和蜂鸣器鸣响进行声光报警，同时在 OLED 显示屏上实时展示环境数据和报警状态。

系统具有启动自检、多传感器融合判断、声光双重报警等功能特点，软件采用模块化设计，具有良好的可扩展性和可维护性。
